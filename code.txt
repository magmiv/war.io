Клиент:

1) Network
   - Принимает данные от сервера:
      - Игроки > Солдаты (только те, что не в тумане войны), мертвые солдаты
      - Строения
   - При получении данных:
      - Обновляет Troops
      - Обновляет строения
      * Чтобы лишный раз не обновлять объекты, можно добавить для них поле state, которое будет приходить в запросе. Если 
         состояние изменилось - пересоздать объекты

2) Troops
   - draw. Отрисовывает только тех солдат, которые находятся в кадре. Каждый солдат имеет состояние (идет, стреляет, ...)
   - update. Принимает зашифрованный массив данных, руководствуясь которыми обновляет все войска на карте. 
      Массив содержит в себе мертвых солдат. Отрисовывет анимацию смерти.

3) MainPlayer
   - Наследуется от всех классов

4) MainPlayerEvents
   - Занимается обработкой кликов по экрану

5) MainPlayerNetwork
   - Отсылает все запросы:
      - newPlayer. После этого клиент начинает получать данные о игровом состоянии, которые обрабатываются в Network
      - Изменение движения у дивизии. Рассчитывает moveTo для каждого солдата и отсылает массив на сервер
      - Строительство здания

6) Player
   - draw
   - id
   - update

7) Division
   - Хранит список солдат

8) Solder
   - Имеет анимацию, хп, moveTo, все такое.


Сервер:

1) Main
   - Хранит Troops
   - Хранит здания
   - 20 раз в секунду отсылает данные на клиенты
   - 60 раз в секунду обновляет Troops
   - Наследуется от Network

2) Network
   - Принимает все запросы от клиентов, обрабатывает их

3) Troops
   - update. Выполняет всю логику для всех игроков

4) Player
   - Id
   - updateMoveToForDivision

5) Division
   - Id
